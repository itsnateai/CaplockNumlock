## Project Summary

CapsNumTray is a lightweight AutoHotkey v2 utility that adds independent system tray icons showing the real-time state of Caps Lock, Num Lock, and Scroll Lock on Windows. The user can left-click any icon to toggle the corresponding key, right-click for a context menu, and configure all options via a Settings GUI. Visibility preferences persist via INI file. Built with raw Win32 `Shell_NotifyIconW` to support multiple independent tray icons — AHK's built-in tray API only supports one.

## Key Features

- Independent tray icons for Caps Lock (ID 10), Num Lock (ID 11), and Scroll Lock (ID 12, opt-in)
- Left-click to toggle, right-click for full context menu
- Settings GUI with GitHub and Help buttons, OK/Apply/Cancel
- Resizable Help window with full usage guide
- OSD floating tooltip on toggle (configurable)
- Optional beep feedback on toggle
- Run at startup via registry `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
- Per-monitor DPI scaling via `GetDpiForWindow(A_ScriptHwnd)`
- Dark/light theme detection via `SystemUsesLightTheme` registry key
- 3-stage icon fallback: disk file → embedded PE resource (compiled) → Windows built-in
- Both-icons-hidden guard — refuses to hide last visible icon with ToolTip warning
- Icons embedded as PE resources via `@Ahk2Exe-AddResource` (IDs 210–215) — no FileInstall

## Lessons Learned

### AHK-Specific

- **Compound statement bug**: In AHK v2, `{statement1 ; statement2}` treats `;` as a comment delimiter, not a statement separator. Any second statement inside braces after `;` is silently ignored. Always put each statement on its own line. Caused P2-1 (SyncIcons silently no-op'd).

- **64-bit struct offsets**: `NOTIFYICONDATA` struct layout is only valid on 64-bit AHK. Without `#Requires AutoHotkey v2.0 64-bit`, running under 32-bit AHK causes silent memory corruption with no error. Always add the 64-bit requirement when using raw Win32 structs with pointer-sized fields.

- **Registry startup path**: Writing a bare `.ahk` path to the Run key fails at login if AHK isn't the default `.ahk` handler. Correct format: `"AhkPath" "ScriptPath"` for script mode, bare path for compiled `.exe`.

- **Global declaration scope**: `global` with no names in a function makes every local variable in that function global — including future typos. A misspelled global name silently creates a new global instead of erroring. Always use `global varA, varB, varC` (explicit names).

- **GUI destroy safety**: Always wrap `Gui.Destroy()` in a try block. If the GUI was already destroyed (user closed it), calling Destroy again throws an unhandled error. Pattern: `CloseXxxGui()` function with `try { g_gui.Destroy() }`.

- **NOTIFYICON_VERSION_4**: Must call `NIM_SETVERSION` after each `NIM_ADD` to enable modern tray protocol. Without it, click coordinates in wParam are wrong on multi-monitor setups. Also requires `NIF_SHOWTIP` flag (`0x80`) for hover tooltips to display correctly.

- **Blank icon flash**: On `NIM_ADD`, send only `NIF_MESSAGE` (no icon/tip yet). Then `NIM_MODIFY` to set icon and tip. Sending all flags together causes a visible blank-icon flash before the real icon loads.

- **@Ahk2Exe-AddResource**: The `;@Ahk2Exe-` prefix makes it look like an AHK comment but the compiler reads it. Use this to embed icons into the .exe at compile time. Access at runtime with `TraySetIcon(A_ScriptFullPath, -resourceId, true)` or via `LoadIco`'s PE resource stage.

- **Icon handle ownership**: Track which handles came from `LoadImage` (owned, must call `DestroyIcon`) vs shared system icons (must NOT call `DestroyIcon` — they're shared GDI objects). Use a `Map` keyed by handle to track ownership.
